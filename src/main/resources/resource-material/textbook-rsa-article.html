Details Behind RSA (Rivest-Shamir-Adleman)

What is RSA?

RSA, short for Rivest-Shamir-Adleman, is an asymmetric public key encryption and decryption scheme that is used to transmit data privately between two parties. This approach differs from symmetric cryptography, where one key is used to encrypt and decrypt a message, whereas RSA uses two keys (a public key for encryption and a private key for decryption), which allows for secure communication between two parties without needing to share an encryption key beforehand. In the mid-1970s, the concept of asymmetric cryptography was proposed by Merkle, Diffie, and Hellman, which allowed parties to exchange public keys. RSA followed this idea and was invented by Rivest, Shamir, and Adleman in 1977. Beyond digital signatures and authentication, RSA can found be in platforms that deal with secure file sharing, secure email communications, etc.

The Run-down of How Textbook RSA Operates

Quick Overview

This article will only focus on textbook RSA. The term "textbook RSA" just gives you the basic math run-down of how RSA operates, and Cipher-Labs RSA module follows the principles of textbook RSA for educational purposes only. Textbook RSA should never be used in the real world. RSA consists of three general steps: generating keys, encrypting messages, and decrypting messages.

Scenario

Now let's assume Bob wished to send the message "CAT" to Alice. To accomplish this task, the three major steps of RSA must be fulfilled.

Key Generation

The key generation process follows these given steps.

  1. The first step involves coming up with two large random primes, and these values are known as p and q. Never share the values of p and q with anybody! In the real world, p and q should be so large that they should be impossible to factor, like 21023 or 21024 which equates to a 2048-bit length key. For this example, p will be 8009, andq will be 8011.
  2. Next step involves computing N. N is found by finding the product of p and q. N is used as the modulus for public and private keys.
    • N = (p * q)
    • N = (8009*8011)
    • N = 64,160,099
  3. Compute phi, or Euler's totient of N (φ(N)). phi can be found using the following formula (p-1) * (q - 1). Putting it all together it would look like this:
    • phi = (p-1) * (q - 1)
    • phi = (8009-1) * (8011-1)
    • phi = 64,144,080
  4. Choose an appropriate integer value for e. The letter e is known as the public exponent. Before assigning e a value, two rules must be met. e must be greater than one, and coprime to phi or 1 < e < φ(N). In this case e will be assigned the value 17.
  5. Determine your private key by finding d as d = e-1 (mod(φ(N)). d is the modular multiplicative inverse of e modulo φ(N). The extended Euclidean algorithm can be used to find d, but since we are working with smaller numbers and to keep things simple, we'll use a multiplier known as k. First create a four column table, where column 1 contains k, column 2 contains phi*k, column 3 contains (phi * k) + 1, and column 4 will contain ((phi * k) + 1)/e. Until we answer the question, if ((phi*k) + 1) / e results in a whole number, that'll conclude what our private key is. Make sure to increment k by 1 in each new row.
    6. k Value phi * k (phi * k) + 1 ((phi * k) + 1) / e
    1 64,144,080 64,144,081 3,773,181.235
    2 128,288,160 128,288,161 7,546,362.412
    3 192,432,240 192,432,241 11,319,543.59
    ... ... ... ...
    11 705,584,880 705,584,881 41,504,993
  6. Our private key is 41,504,993 since it's a whole number. To verify if d's value is correct simply just use this formula:
    • (e*d) - (k*phi) = 1
    • (17*41,504,993) - (11 * 64,144,080) = 1
    • After successfully verifying, d's value is in fact 41,504,993

After solving to get the appropriate public keys and private key, Alice's public key is 64,144,080 (φ(N)), and 17 (e), while her private key is 41,504,993 (d).

Encrypting A Message

Once Bob, obtains Alice's public key, Bob is capable of sending the message "CAT" to her securely. Before jumping into the encryption process, Bob first needs to convert the word "CAT" into a numbered value. Bob can easily encode each character in the text "CAT" to its appropriate ASCII decimal value.

  1. C = 67
  2. A = 65
  3. T = 84
  4. Byte Sequence Result: [67, 65, 84] and the total number of bytes is 3.

With the byte sequence ready, we convert it into a large integer using the base-256 conversion formula which will give us M (our message). We have a 3 byte sequence, the formula would look something like this:

  • M = (B1 * 2562) + (B2 * 2561) + (B3 * 2560)
  • M = (67 * 2562) + (65 * 2561) + (84 * 2560)
  • M = 4,407,636
  • Note since M < N, we can appropriately encrypt the following message "CAT"

The formula for encryption is C = Me (mod N)

  1. C is the ciphertext
  2. M is the plaintext
  3. e is the public exponent also the recipient's public key
  4. mod N is modulus N and N is the recipient's public key
  • C = Me (mod N)
  • C = (4,407,636 17) (mod 64,160,099)
  • C = 16,575,496

On Alice’s end, she would receive the encrypted message “16,575,496” from Bob. To read this as plaintext, Alice would need to use her private key to decrypt it.

Decrypting A Message

The formula for decryption is M = Cd (mod N)

  1. M is the plaintext
  2. C is the ciphertext
  3. d is the given private key
  4. mod N is modulus N and N is the Alice's public key

Putting the formula all together:

  • M = Cd (mod N)
  • M = 16,575,49641,504,993 (mod 64,160,099)
  • M = 4,407,636

Remember we converted our message into Base-256 representation. If Alice want's to see what the original text is, she is going have to figure out the length of the byte sequence. Alice can simply divide 4,407,636 / 256, and repeat this step until she reaches 0, to reveal what the message is. Remainders will reveal what the message is.

  1. 4,407,636/256 = 17,217 with a remainder of 84
  2. 17,217/256 = 67 with a remainder of 65
  3. 67/256 = 0 with a remainder of 67

Remember to reverse the order of the characters found, since we are working backwards. From there, convert each remainder value to their respective ASCII character. [67, 65, 84] gives us [C, A, T]. Alice successfully got the plaintext message "CAT" from Bob.

Security Implications

Textbook RSA is never meant to be used in live production systems, and is meant to be used for educational purposes only. Textbook RSA lacks padding mechanics, leading to RSA encryption being purely deterministic, and this crypto system is vulnerable to man-in-the-middle attacks, chosen plaintext attack, and cube root attacks (if e was 3, for example). These are some attacks that come to mind, but in real-world applications, RSA 2048-bit is primarily used as it still provides security, and would take billions of years for even super computers to crack, though quantum technology is on the rise, lowering the time to potentially crack 2048-bit key spaces.
