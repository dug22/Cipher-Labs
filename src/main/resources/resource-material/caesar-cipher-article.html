Details Behind the Caesar Cipher

What is the Caesar Cipher?

A Caesar cipher is a simple monoalphabetic substitution cipher, where each character within the given plaintext is shifted a specific number of times, which is known as the key. For example, with a positive shift of 3, the letter "C" would become "F". This method of this cipher was named after Julius Caesar, in his honor.

The Encryption Process

The encryption process of Caesar's Cipher involves substituting each character to the right, within the plaintext with a given number (known as the key). For example, let's assume Bob wished to send Alice the following message "THIS IS A SECRET", with a shift of 3. T would become W, H would become K, I would become L, and so on. The final result would be:

Plaintext: THIS IS A SECRET

Ciphertext: WKLV LV D VHFUHW

If you need a visual representation to help you out with this, you can write a 2x26 table containing the plaintext alphabet in chronological order, followed by the ciphertext letters with a shift of 3. Do note that these tables will always be different depending on your shift's value.

Plaintext ABCDEFGHIJKLMNOPQRSTUVWXYZ
Cipher (Shift 3) DEFGHIJKLMNOPQRSTUVWXYZABC

The encryption can be represented using modular arithmetic, by mapping the appropriate letters into numbers, matching the scheme, A=0, B=1, ..., Z=25. The formula is as followed:

E(x) = (x + n) mod 26

  • The value x resembles the plaintexts character position within the alphabet, ranging from 0-25.
  • The value n resembles the shift's value (your key).
  • "mod 26" resembles the modulo operation, and it ensures the shift "wraps around" to the beginning of the alphabet if it passes Z.

The Decryption Process

The decryption process of the Caesar cipher follows the same idea as the encryption process, except you shift each character in the ciphertext backwards instead. Let's assume Alice wished to read Bob's following message "WKLV LV D VHFUHW" in plaintext. She would take the following ciphertext with its key, and shift the letters back 3. W would become T, K would become H, L would become I, and so on. The final result would be:

Ciphertext: WKLV LV D VHFUHW

Plaintext: THIS IS A SECRET

If you need a visual representation to help you out with this, you can write a 2x26 table containing the ciphertext letters with a shift of 3, followed by the plaintext alphabet in chronological order.

Cipher (Shift 3) DEFGHIJKLMNOPQRSTUVWXYZABC
Plaintext ABCDEFGHIJKLMNOPQRSTUVWXYZ

The decryption can be represented using modular arithmetic, by mapping the appropriate letters into numbers, matching the scheme, A=0, B=1, ..., Z=25. The formula is as followed:

D(x) = (x - n) mod 26

  • The value x resembles the plaintexts character position within the alphabet, ranging from 0-25.
  • The value n resembles the shift's value (your key).
  • "mod 26" resembles the modulo operation, and it ensures the shift "wraps around" to the beginning of the alphabet if it passes Z.

Security Implications

Caesar's cipher has many security implications, making it a weak cipher. There are 26 possible keys the user could use to attempt to crack this cipher. A simple brute force attack of 26 possible key shifts can easily crack a message that was encrypted using the Caesar Cipher, in mere seconds.
